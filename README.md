## MyTree是一个种树的小游戏

一个初学者的C++项目

# 项目文档 - aTree

## 项目概述

一个简单的命令行种树游戏，玩家通过浇水、施肥等操作培养一棵树苗成长为大树。游戏完全使用C++实现

###### PS.争取开发基于面向对象和面向过程两种不同编程模式的两个版本（功能一致）

## 游戏目标

- 将一棵小树苗培养成参天大树
- 体验植物成长的过程
- 学习基本的C++编程概念

## 基础需求（必须实现）

### 1. **游戏状态管理系统**

- 树的属性追踪：
  - 大小/等级 (1-10级)
  - 水分值 (0-10)
  - 营养值 (0-10)
  - 健康度 (0-20，降到0游戏结束)
  - 当前天数
- 游戏状态判断：
  - 胜利条件：树达到最大尺寸(10级)
  - 失败条件：健康度降到0

### 2. **用户交互界面**

- 主菜单显示：

  ```
  1. 浇水
  2. 施肥
  3. 查看状态
  4. 等待一天
  5. 退出游戏
  ```

  

- 清晰的游戏状态展示（文本格式）

- 操作反馈信息（成功/失败提示）

### 3. **游戏循环与逻辑**

- 主游戏循环结构：

  ```
  while (1) {
      显示状态
      获取玩家输入
      处理操作
      更新游戏状态
      检查游戏结束条件
  }
  ```

  

- 每日资源消耗机制（水分、营养自然减少）

- 成长条件检查：水分≥4且营养≥4时才能生长

### 4. **核心操作功能**

- 浇水：增加水分值
- 施肥：增加营养值
- 查看状态：显示所有属性
- 等待一天：时间推进，自然消耗发生
- 退出游戏：保存并退出

## 中等需求（建议实现）

### 5. **<mark>随机事件系统</mark>**

- 发生概率：每天约15-20%几率
- 事件类型：
  - 正面事件（增加资源/生长）
  - 负面事件（减少资源/健康）
  - 中性事件（提示信息）
- 示例事件：
  - 下雨：水分大幅增加
  - 阳光明媚：促进生长
  - 虫害：健康度下降
  - 小鸟带来养分：营养增加

### 6. **游戏平衡机制**

- 资源管理：水分/营养的获取与消耗平衡
- 成长节奏控制：不能过快或过慢
- 健康度调节：资源不足时健康下降，资源充足时健康恢复
